# Next.js | Introduction

## Overview

Next.js adalah sebuah framework React yang populer dan kuat yang digunakan untuk membangun aplikasi web modern. Dikembangkan oleh tim Vercel, Next.js menawarkan berbagai fitur canggih yang membuat pengembangan aplikasi web menjadi lebih efisien dan produktif.

Dengan menggunakan Next.js, kamu dapat dengan mudah membuat aplikasi web dengan performa tinggi dan pengalaman pengguna yang luar biasa. Framework ini telah menjadi pilihan utama bagi banyak pengembang karena kemampuannya dalam menghadapi tuntutan kompleksitas proyek modern.

Next.js dirancang untuk memberikan solusi atas beberapa masalah umum dalam pengembangan aplikasi web, seperti rendering server-side, routing dinamis, code splitting, pre-rendering halaman, dan masih banyak lagi. Dalam artikel ini, kita akan menjelajahi fitur utama dari Next.js serta keuntungan dan kekurangan penggunaannya.

## Apa itu Next.js?

Next.js adalah sebuah framework JavaScript yang bersifat open-source dan digunakan untuk mengembangkan aplikasi web modern. Dikembangkan oleh tim Vercel, Next.js memberikan kemudahan dalam membangun aplikasi dengan menggunakan React, serta menghadirkan berbagai fitur yang sangat berguna bagi pengembang.

Dengan Next.js, kita dapat membuat aplikasi web yang responsif dan cepat, dengan dukungan server-side rendering (SSR) dan static site generation (SSG). SSR memungkinkan konten dari halaman web disajikan langsung dari server sebelum dikirim ke browser pengguna. Sementara itu, SSG memungkinkan kita untuk menghasilkan halaman statis saat waktu pembuatan build, sehingga meningkatkan kecepatan akses dan kinerja aplikasi.

Next.js juga menyediakan fitur hot module replacement (HMR) yang memungkinkan perubahan kode JavaScript langsung terlihat tanpa perlu melakukan refresh pada browser. Fitur ini sangat membantu dalam proses pengembangan karena dapat menghemat waktu dan meningkatkan produktivitas.

Dalam mengembangkan aplikasi dengan Next.js, kita juga akan mendapatkan kelebihan dari konfigurasi default yang sudah disediakan oleh framework ini. Misalnya, konfigurasi routing otomatis sehingga tidak perlu melakukan setting secara manual untuk setiap halaman baru yang dibuat. Selain itu, Next.js secara otomatis melakukan code-splitting sehingga hanya kode yang diperlukan pada halaman tersebut saja yang akan diunduh oleh browser pengguna.

## Apa yang Membedakan Next.js Dengan React?

Next.js adalah framework React yang dikembangkan dengan tujuan untuk mempermudah proses pengembangan aplikasi web. Meskipun Next.js menggunakan React sebagai dasarnya, ada beberapa perbedaan signifikan antara keduanya.

Pertama-tama, Next.js menyediakan server-side rendering (SSR) secara default. SSR merupakan teknik untuk menghasilkan HTML pada sisi server sebelum dikirim ke browser. Hal ini memungkinkan konten website dapat diakses lebih cepat oleh pengguna dan juga mendukung SEO yang lebih baik. Berbeda dengan React yang melakukan rendering di sisi klien (client-side rendering), Next.js menggabungkan SSR dan client-side rendering sehingga memberikan pengalaman pengguna yang lebih baik.

Selain itu, routing di Next.js juga lebih sederhana dibandingkan dengan React. Dalam React, kita perlu mengimplementasikan routing secara manual menggunakan library tambahan seperti react-router. Namun dalam Next.js, kita dapat menggunakan fitur built-in untuk menentukan halaman dan rute aplikasi dengan mudah hanya melalui struktur berkas dan folder.

Fitur lain yang membedakan Next.js dengan React adalah kemampuan untuk melakukan pre-fetching data (data pre-fetching). Saat menjelajahi sebuah halaman di website yang dibangun dengan Next.js, data yang diperlukan akan dipre-fetch sebelum halaman tersebut ditampilkan kepada pengguna. Hal ini meningkatkan kecepatan loading halaman dan memberikan pengalaman browsing yang lebih lancar pada user.

## Kelebihan dari Next.js

Next.js memiliki sejumlah kelebihan yang membuatnya menjadi pilihan utama dalam pengembangan aplikasi web React.

1. **Server-Side Rendering (SSR)**:
   Next.js menyediakan dukungan bawaan untuk SSR, memungkinkan rendering halaman di server sebelum dikirim ke browser. Hal ini meningkatkan performa aplikasi dengan mengurangi waktu pemuatan halaman, karena kontennya sudah ada sebelum tampil di layar pengguna.

2. **Static Site Generation (SSG)**:
   SSG memungkinkan kita menghasilkan halaman statis pada waktu pembangunan. Ini sangat bermanfaat untuk meningkatkan kecepatan pemuatan halaman dan mengurangi beban server, terutama untuk halaman yang jarang berubah.

3. **Automatic Code Splitting**:
   Next.js secara otomatis memecah kode menjadi bundle yang lebih kecil, yang hanya diunduh jika diperlukan. Ini mengoptimalkan waktu pemuatan halaman dengan mengurangi jumlah data yang harus diunduh pengguna.

4. **Hot Module Replacement (HMR)**:
   HMR memungkinkan kita melihat perubahan langsung saat kita mengedit kode tanpa perlu menyegarkan halaman secara manual. Ini membuat siklus pengembangan menjadi lebih cepat dan efisien.

5. **Peningkatan SEO**:
   SSR membantu meningkatkan indeksabilitas mesin pencari karena konten halaman dihasilkan di server. Ini berkontribusi pada peringkat SEO yang lebih baik.

6. **Rich Ecosystem**:
   Next.js memiliki ekosistem yang kaya dengan banyak plugin dan dukungan komunitas yang besar. Hal ini membuatnya lebih mudah untuk menemukan solusi dan alat bantu yang diperlukan dalam pengembangan.

## Kekurangan dari Next.js

Setiap teknologi memiliki kelebihan dan kekurangan, begitu juga dengan Next.js. Meskipun memiliki banyak keunggulan, ada beberapa kekurangan yang perlu diperhatikan sebelum menggunakan Next.js dalam pengembangan aplikasi web.

1. **Kurva Pembelajaran Awal**:
   Bagi pengembang yang baru memulai, Next.js mungkin memiliki kurva pembelajaran yang agak curam, terutama jika belum memiliki pengalaman dengan konsep-konsep SSR dan SSG.

2. **Kompleksitas SSR**:
   Meskipun SSR meningkatkan performa, implementasinya dapat menjadi kompleks terutama pada proyek-proyek besar. Pengembang perlu memahami dengan baik arsitektur aplikasi dan manajemen state untuk menghindari potensi kerumitan.

3. **Ukuran Build yang Besar**:
   Secara default, build Next.js bisa menjadi cukup besar, terutama jika proyek memiliki banyak dependensi. Hal ini bisa berdampak pada waktu yang dibutuhkan untuk melakukan deploy.

4. **Pengaturan Tambahan untuk SSR/SSG**:
   Beberapa proyek mungkin memerlukan konfigurasi tambahan untuk SSR atau SSG, terutama jika ada kebutuhan yang lebih kompleks. Ini bisa memerlukan pemahaman yang lebih mendalam tentang konfigurasi Next.js.

5. **Keterbatasan dalam SSR Eksternal**:
   Meskipun Next.js sangat baik dalam SSR halaman yang dibuat dengan frameworknya sendiri, SSR untuk aplikasi yang menggunakan server backend eksternal mungkin memerlukan penanganan khusus.

## Cara Instalasi

Langkah pertama untuk menggunakan Next.js adalah menginstalnya. Kita dapat melakukan instalasi menggunakan npm atau yarn. Berikut adalah langkah-langkahnya:

Syarat-syarat:

- Versi [Node.js 18.17](https://nodejs.org/) atau diatasnya.
- macOS, Windows (termasuk WSL), dan Linux.

Pada terminal yang digunakan, jalankan perintah dibawah untuk men-generate project,

```bash copy
npx create-next-app@latest
```

Lalu setelah perintah diatas nantinya akan ada interaksi didalam terminal untuk memilih konfigurasi daripada project Next.js kita nantinya

```bash
1. What is your project named? my-app
2. Would you like to use TypeScript? No / Yes
3. Would you like to use ESLint? No / Yes
4. Would you like to use Tailwind CSS? No / Yes
5. Would you like to use 'src/' directory? No / Yes
6. Would you like to use App Router? (recommended) No / Yes
7. Would you like to customize the default import alias (@/*)? No / Yes
8. What import alias would you like configured? @/*
```

Setelah proses pemilihan konfigurasi selesai, nantinya `create-next-app` akan membuat suatu folder bernama sesuai dari nama project yang telah kita pilih sebelumnya.

## Menjalankan Server Development

1. Jalankan perintah `npm run dev` pada terminal IDE untuk menjalankan server development.
2. Buka http://localhost:3000 pada peramban yang digunakan untuk melihat web app kita.
3. Ubah file `app/page.tsx` (atau `pages/index.tsx`) lalu simpan untuk melihat perubahan tampilan/code secara langsung.

## Kesimpulan

Next.js memberikan pendekatan yang inovatif dalam membangun aplikasi web dengan mengintegrasikan fitur-fitur penting seperti server-side rendering, static site generation, dan routing.

Dibandingkan dengan React murni, Next.js menawarkan beberapa kelebihan yang signifikan. Fitur server-side rendering memungkinkan aplikasi web untuk lebih cepat dimuat dan memberikan pengalaman yang lebih baik kepada pengguna. Selain itu, static site generation memungkinkan pembangunan halaman statis secara otomatis berdasarkan data yang diberikan, sehingga meningkatkan performa dan skala aplikasi.

Next.js juga menawarkan kemudahan dalam hal routing. Framework ini menyediakan sistem routing internal yang otomatis menghasilkan URL yang ramah SEO serta memiliki dukungan penuh untuk dynamic routes. Dengan adanya fitur-fitur ini, developer dapat fokus pada pengembangan aplikasi tanpa harus khawatir tentang konfigurasi routing secara manual.

Secara keseluruhan, Next.js adalah framework yang kuat dan sangat berguna bagi para pengembang web. Dengan kemampuannya dalam melakukan server-side rendering serta static site generation, membuat aplikasi menjadi lebih cepat dimuat dan memiliki performa yang unggul tanpa harus melakukan banyak konfigurasi tambahan. Jika kita ingin meningkatkan kualitas dan performa website kita dengan mudah menggunakan React, maka Next.js adalah pilihan terbaik.
